# Библиотека индексации текстовых файлов

Библиотека для индексации текстовых файлов по словам с поддержкой
многопоточного доступа и автоматического отслеживания изменений.

## Возможности

- Индексация файлов и каталогов (включая вложенные)
- Поиск файлов по слову
- Автоматическое отслеживание изменений с помощью `WatchService`
- Потокобезопасный доступ через `ReentrantReadWriteLock`
- Расширяемая токенизация и чтение файлов

## Технологии

- Java 21
- JUnit 5
- Gradle
- Lombok

## Запуск
### Для корректного отображения русского языка перед запуском введите
```bash
chcp 65001
```

#### Билд
```bash
./gradlew build
```
#### Запуск
```bash
./gradlew run
```
#### Тесты
```bash
./gradlew test
```
Для проверки программы достаточно запустить. Демонстрационная CLI программа написана и описана. В консоли отобразится вся нужная информация. Можно ввести `lib help` для просмотра доступных команд.

## Пример использования

```java
try (Library library = new Library()) {
    library.indexDirectory(Path.of("/docs"));
    Set<Path> results = library.search("кошка");
}
```
*try-with-resources* для автоматического закрытия внутреннего `WatchService`.

## Архитектура

| Компонент       | Ответственность      |
|-----------------|----------------------|
| `Library`       | API                  |
| `InMemoryIndex` | Хранение индекса     |
| `FileProcessor` | Чтение и токенизация |
| `FileMonitor`   | Слежение за файлами  |

## Принятые решения

| Решение                       | Причина                                                                                                                                                                       |
|-------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `ReadWriteLock`               | Параллельное чтение без блокировок                                                                                                                                            |
| `AutoCloseable`               | Корректное освобождение ресурсов                                                                                                                                              |
| Контракт `Tokenizer`          | Расширяемость. Новый механизм разделения реализует контракт и свою функцию разделения                                                                                         |
| Контракт `TextReader`         | Расширяемость. Для бинарных файлов (типа *.pdf* и *.docx* нужны отдельные сторонние библиотеки. Для чтения таких файлов необходимо реализовать контракт и свою функцию чтения |
| Пути переводятся в абсолютные | Безопасность и удобство. Все структуры в программе работают с абсолютными файлами                                                                                             |
